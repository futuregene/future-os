# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

Rust (agent) + TypeScript (TUI). The Rust binary is the backend; the TS TUI provides the terminal interface.

## Build/Run/Test

Prefer `make` targets from repo root. For more control, use cargo/npm directly.

```bash
# ─── Make targets (from repo root) ──────────────────────────────────────────
make build              # Build both agent and TUI
make build-agent        # Build Rust agent only
make build-tui          # Build TypeScript TUI only
make test               # Run all Rust tests
make lint               # Lint Rust (fmt+clippy) and TypeScript (tsc --noEmit)
make lint-agent         # cargo fmt --check && cargo clippy
make lint-tui           # npx tsc --noEmit
make fmt                # Format Rust code with cargo fmt
make run-agent          # Build and run Rust agent
make run-tui            # Run TUI in dev mode (auto-installs npm deps)
make clean              # Remove target/, dist/, node_modules/

# ─── Direct commands ────────────────────────────────────────────────────────
cd agent && cargo run                  # Build and run (starts gRPC server on 127.0.0.1:50051)
cd agent && cargo run -- --model deepseek-v4-flash "hello"  # Non-interactive prompt
cd agent && cargo test                 # Run all Rust tests
cd tui && npm run dev                  # Run TUI directly with bun (no pre-build)

# Proto codegen — regenerated by agent/build.rs on every cargo build
# Proto source: proto/proto/future.proto → agent/src/grpc/generated/proto.rs
```

The Rust binary (`future-agent`) is the backend. The TS TUI connects to it via gRPC. When launched with no messages, the Rust binary starts a gRPC server on `127.0.0.1:50051` and the TUI connects to it.

Config is under `~/.future/agent/` (settings.json, models.json, auth.json, sessions/) for agent backend, and `~/.future/tui/` (settings.json) for TUI preferences. The agent's settings.json holds model/provider defaults and skills config; TUI settings.json holds theme, terminal, keybinding preferences.

## Architecture

```
                  ┌──────────────────────────┐
                  │   Rust Agent Backend     │
                  │   (agent/ — gRPC server) │
                  └──────────┬───────────────┘
                             │ gRPC (tonic)
                  ┌──────────┴───────────────┐
                  │                         │
           TypeScript TUI             Print mode
           (tui/ via gRPC)         (--print flag)
```

### Rust backend (`agent/src/`)

Entry point: `main.rs` — CLI arg parsing (clap), model/provider/auth resolution, then either starts gRPC server or runs a single prompt non-interactively.

| Module | Role |
|--------|------|
| `agent/mod.rs` | Agentic loop: call LLM → stream events → execute tools → repeat. Supports interrupt/abort, steering/followUp queues, parallel tool execution, auto-retry with exponential backoff |
| `engine/mod.rs` | `Engine` struct: wires provider, tools, session, and agent loop together |
| `llm/mod.rs` | OpenAI-compatible streaming HTTP client (reqwest + SSE parsing). Supports thinking/reasoning content extraction, tool call accumulation from streaming chunks |
| `grpc/mod.rs` | gRPC server using tonic. Implements `FutureAgent` service: `ExecuteCommand` (unary), `StreamEvents` (server-side streaming) |
| `rpc/mod.rs` | Command handler dispatch (25+ commands: prompt, steer, abort, set_model, compact, fork, etc.) and `ServerSession` state management |
| `session/mod.rs` | Conversation persistence as JSONL files in `~/.future/agent/sessions/<encoded-cwd>/`. Tree-structured entries with ParentID for forks. `Manager` handles save/load/list |
| `types/mod.rs` | Core types: `Message`, `StreamEvent`, `AgentTool`, `ToolDef`, `AgentConfig`, `LLMProvider` trait, `ContentBlock` (polymorphic text/image/tool_result), and `ConvertToLLM`/`ConvertFromLLM` conversion |
| `tools/mod.rs` | 6 tools: bash, read, write, edit, grep, ls. `coding_tools()` returns default 4 (read, bash, edit, write), `readonly_tools()` returns search tools, `all_tools()` returns all 6 |
| `compaction/mod.rs` | Context compaction: estimates tokens (chars/4 heuristic), finds safe cut points, summarizes file ops |
| `models/mod.rs` | Model registry: built-in catalog + user `models.json`, resolution, fuzzy matching. Default model from settings.json |
| `config/mod.rs` | Settings struct (50+ fields) with deep-merge. Loads from `~/.future/agent/settings.json` (global) and `.future/agent/settings.json` (project) |
| `auth/mod.rs` | Reads API credentials from `~/.future/agent/auth.json` or `~/.future/agent-app/auth.json`, keyed by provider |
| `skills/mod.rs` | Discovers skills from `~/.future/agent/skills/`, `.future/agent/skills/`, `~/.agents/skills/` — parses YAML frontmatter from SKILL.md |
| `prompt/mod.rs` | Builds system prompt from identity, project context (CLAUDE.md/AGENTS.md), skills, and metadata |
| `events/mod.rs` | `EventBus` with pub/sub for bridging agent streaming events to frontends (thinking, tool calls, usage stats) |
| `utils/mod.rs` | Utilities: session ID generation (timestamp+random), cwd path encoding (base32), image MIME type detection from file headers |

Provider model: `LLMProvider` trait (`stream_chat`). Uses OpenAI-compatible HTTP+SSE — no Anthropic-specific client yet. Thinking/reasoning extraction handled via compat format parameters (deepseek, openrouter, zai, qwen).

API key resolution order: `--api-key` flag → `LLM_API_KEY` env → `ANTHROPIC_API_KEY` env → `OPENAI_API_KEY` env → `auth.json` (by provider) → `auth.json` default key.

Session files are JSONL, tree-structured (each entry has ID + optional ParentID). Entry types: session_info, user, assistant, tool, compaction, model_change, thinking_level_change, branch_summary, label, custom.

### gRPC API (`proto/proto/future.proto`)

The `FutureAgent` service has two RPCs:
- `ExecuteCommand(RpcCommand) → RpcResponse` — unary, 30+ command types
- `StreamEvents(StreamRequest) → stream StreamEvent` — server-side streaming for SSE events

Commands include: prompt, steer, followUp, abort, new_session, set_model, set_thinking_level, compact, fork, clone, export_html, cycle_model, cycle_thinking_level, list_sessions, switch_session, get_available_models, and more.

### TUI architecture (`tui/src/`)

Follows a `Component`/`Container`/`Focusable` pattern with overlay stack:

**Core framework** (`tui.ts`): `Component` interface (render/input/invalidation), `Container` class (addChild/removeChild, cascading invalidation), `OverlayHandle`, `InputListener` pipeline, `NodeTerminal` for raw-mode stdin with Kitty CSI-u, bracketed paste, SGR mouse tracking.

**Rendering** (`app.ts`): Dual-phase scheduling (nextTick + setTimeout at ~60fps), differential rendering (changed-line ranges → minimal ANSI updates), overlay compositing, Kitty image lifecycle management.

**Key systems**: `keys.ts` (Kitty CSI-u/xterm/legacy parsing), `keybindings.ts` (KeybindingManager with context filtering), `theme.ts` (256-color + style composers), `utils.ts` (Intl.Segmenter-based visibleWidth, ANSI-aware word wrap, CJK/emoji grapheme width).

**16 components**: ChatArea, Editor, Footer, MarkdownRenderer, AutocompleteManager, SelectList, SettingsList, Image, Box, Text, TruncatedText, Spacer, Loader, CancellableLoader, Container.

**RPC** (`rpc/`): gRPC client (`grpc-client.ts`) connecting to the Rust backend. Methods: prompt, abort, getState, getAvailableModels, setModel, cycleModel, cycleThinkingLevel, listSessions, switchSession, newSession, compact.
