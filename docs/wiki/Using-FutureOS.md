# Using FutureOS

This page is a tour of the app — the layout, the two kinds of conversations, and how to follow and steer the agent's work.

## The three panels

FutureOS has a three-column layout:

```
   Left sidebar          Conversation           Right panel
   navigation            (the chat)             (context)
```

- **Left — navigation.** New Chat, Research, Data, Skill, your Workspaces and their conversations, your Chats, and Settings. You can expand, collapse, and archive items here.
- **Middle — the conversation.** Your messages and the agent's streaming replies, plans, tool activity, command previews, errors, and approval prompts. The message box stays pinned at the bottom.
- **Right — context.** Inspect what the agent is doing without it taking over your screen. You can collapse this panel when you don't need it.

## Chats vs. Workspaces

A **conversation** in FutureOS is a piece of work you can return to, not a throwaway log. There are two kinds:

| | **Chat** | **Workspace conversation** |
|---|---|---|
| Setup | Instant — starts a temporary working area for you | You pick a folder on your computer |
| Best for | Quick questions, one-off tasks | Real projects and working with your files |
| Right panel shows | Runs, Artifacts | Runs, Review |

Each conversation runs as its own independent agent session, so different chats never get their work tangled together.

## Talking to the agent

Type in the box at the bottom and send. While the agent works you'll see text stream in, along with its plan, the tools it calls, and short summaries of background activity.

- **Model selector** sits in the message box — switch models per conversation.
- **Attachments:** add images with the attach button, by pasting, or by dragging files in — up to 4 per turn.

## Approvals — you stay in control

When the agent wants to do something risky, it **stops and asks first**. An approval card appears just above the message box and the agent waits — it won't time out.

- **Allow** → the agent continues.
- **Reject** → that action is cancelled and the agent is told, so it can adjust.

This is how FutureOS lets the agent do real work while keeping you in charge.

## The right panel: checking the work

- **Runs** — background tasks, split into running and finished. You can stop a running task or clear finished ones. Each card shows the actual command being run.
- **Review** *(Workspace conversations)* — the changes made to your files: a file list, stats, and diffs, so you can see exactly what changed. Folders under version control also offer a "last round of changes" view.
- **Artifacts** *(Chats)* — the files and outputs the conversation produced.

## More in the sidebar

- **Research** — a guided workflow for deep, multi-source research with a written, cited report.
- **Data** — connect and manage data sources.
- **Skill** — browse capability packs the agent can use. See [[Skills]].
