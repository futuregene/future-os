# 使用 FutureOS

这是应用导览:三栏布局、Chat 与 Workspace、和 agent 对话、批准机制,以及如何核对 agent 的工作。

---

## 三栏布局

FutureOS 分为三栏:

- **左侧 —— 导航。** 从上到下依次是:**New Chat**、**Models**(进入设置的快捷入口)、**Skills**、你的 **Workspaces**(每个可展开出它下面的会话)、你的 **Chats**,以及底部的 **Settings**。左栏可折叠,给对话区腾出更多空间。
- **中间 —— 对话区。** 你的消息、流式回复、计划、工具活动、命令预览、错误,以及批准卡片。输入框固定在底部。
- **右侧 —— 上下文面板。** 查看 agent 在做什么(Runs / Review / Artifacts)。可折叠——想核对工作时打开,不需要时收起。

---

## Chat 与 Workspace

每个会话都是独立的 agent 会话,互不干扰。

| | **Chat** | **Workspace** |
|---|---|---|
| 建立方式 | **New Chat** —— 直接开始输入 | **Workspace** —— 打开电脑上的一个文件夹 |
| 适用场景 | 快速提问、一次性任务 | 绑定文件夹的真实项目 |
| 右侧面板显示 | **Runs** 和 **Artifacts** | **Runs** 和 **Review** |

- **Chat** 是最快的提问方式,它的工作区是临时的。
- **Workspace** 绑定一个真实文件夹,agent 会在其中读写文件——**Review** 视图会精确展示改动了什么。

你可以在左栏每个会话旁的菜单里重命名、置顶或删除会话。

---

## 和 agent 对话

- 在输入框输入,按 **Enter** 发送;用 **Shift+Enter** 换行。
- **模型选择器**和**思考级别**控制就在输入框里——可以逐会话切换模型。
- 每条消息最多附 **4 个图片**:用回形针按钮、粘贴,或把文件拖到框上。
- 回复流式输出时,发送按钮会变成**停止**按钮,方便你中断。

## 批准机制 —— 你掌控

这是 FutureOS 的核心。当 agent 想做会产生真实影响的操作——**读写文件、跑 shell 命令、删除文件、或在 workspace 之外写入**——它会**停下来**,在输入框上方弹出一张**批准卡片**,并**不设超时**地等你决定。

卡片会清楚说明请求的内容——要跑的命令、要写的文件(带预览)、或涉及的路径。然后你来选:

- **Allow once(允许一次)** —— 放行这一次操作并继续。
- **Deny(拒绝)** —— 取消该操作。agent 会被告知,以便调整思路、换个做法。
- **Allow in this workspace / this chat(在本项目/本会话允许)**(在可用时) —— 保存一条规则,让本项目里类似的操作不再询问。保存前你可以编辑路径匹配模式。

> 键盘:**Cmd/Ctrl+Enter** 批准,**Esc** 拒绝(若规则编辑器打开,则先关闭它)。

### 批准模式

你可以在**设置 → General** 里、或输入框里的盾牌控件上设置 agent 询问的程度:

- **Manual(手动)** —— 读写文件前询问;只读命令自动运行。
- **Sandboxed(沙箱,仅 macOS)** —— 命令在 macOS 沙箱内运行;文件操作仍会询问。
- **Unrestricted(不受限)** —— 不询问、不沙箱,一切照跑。请谨慎使用。

## 核对工作(右侧面板)

打开右侧上下文面板,在其顶部的下拉里选视图。

### Runs

agent 跑的每个后台任务都会以卡片形式出现,显示**真实命令**、状态,以及运行中/已完成的计数。你可以:

- **Inspect(查看)** 某个任务的细节。
- **Terminate(终止)** 运行中的任务。
- **Clear finished(清理已完成)** 保持整洁。

### Review(Workspace)

对 Workspace,**Review** 视图展示项目里的文件改动:改动文件列表、改动类型(新增 / 修改 / 删除 / 重命名),以及逐文件的 diff。当文件夹处于版本控制下时,你还可以切到**「上一轮改动」**视图,只看最近一次运行改了什么。

### Artifacts(Chat)

对 Chat,**Artifacts** 视图收集 agent 生成的产出——报告、摘要、表格和文件。你可以预览它们、复制内容、导出或打开原始文件。你也可以自己上传文件到 Artifacts。
